# Менеджер Паролей (Password Manager)

Современное кроссплатформенное Desktop-приложение для безопасного хранения и управления паролями. Разработано на **Kotlin Multiplatform** с использованием **Compose Desktop**.

Приложение обеспечивает высокий уровень безопасности благодаря шифрованию **AES-256** на стороне клиента и предоставляет удобный интерфейс в стиле **Material Design 3**.

---

## Основные возможности

### Безопасность
*   **Шифрование:** Все данные шифруются алгоритмом **AES-256** (CBC mode) перед сохранением в базу данных.
*   **Хеширование:** Мастер-пароль защищен алгоритмом **PBKDF2WithHmacSHA256** (10,000 итераций) с уникальной солью.
*   **Zero-Knowledge:** Пароли расшифровываются только в оперативной памяти в момент отображения.

### Функционал
*   **Генератор паролей:** Создание криптостойких паролей с настройкой длины и набора символов.
*   **Управление записями:** Добавление, редактирование, удаление и копирование паролей в буфер обмена.
*   **Тегирование:** Гибкая система тегов для фильтрации (например: "Работа", "Финансы").
*   **Поиск:** Мгновенный поиск по названию, логину или URL.
*   **Импорт/Экспорт:** Резервное копирование базы данных в JSON формат.

## Архитектура проекта

Проект построен на принципах **Layered Architecture** (Многослойная архитектура) с четким разделением ответственности:

### 1. Presentation Layer (Слой представления)
*   **Технология:** Compose Multiplatform.
*   **Паттерн:** Unidirectional Data Flow (UDF). Состояние поднимается вверх к экранам (`DashboardScreen`, `LoginScreen`), а события спускаются вниз.
*   **Компоненты:** `ui/screens`, `ui/components`, `ui/dialogs`.

### 2. Domain Layer (Слой бизнес-логики)
*   **Ответственность:** Правила обработки данных, криптография, модели.
*   **Компоненты:**
    *   `model/Model.kt`: Data-классы и DTO.
    *   `utils/Security.kt`: Реализация алгоритмов шифрования и хеширования.

### 3. Data Layer (Слой данных)
*   **Ответственность:** Персистентное хранение данных.
*   **Технология:** SQLite + Exposed ORM.
*   **Компоненты:** `data/Database.kt` (Управление соединениями и CRUD операции).

## Технологический стек

*   **Язык:** Kotlin 1.9+
*   **UI Framework:** Jetbrains Compose Multiplatform (Desktop)
*   **Database:** SQLite (JDBC Driver)
*   **ORM:** Jetbrains Exposed
*   **Cryptography:** Java Cryptography Architecture (JCA)
*   **Serialization:** Kotlinx Serialization (JSON)
*   **Testing:** JUnit 5, Kotlin Test
*   **Documentation:** Dokka (HTML Generator)
*   **Build System:** Gradle (Kotlin DSL)

## Тестирование

Проект покрыт автоматическими тестами, проверяющими критически важные узлы системы:
1.  **Security Tests:** Проверка уникальности соли, корректность шифрования/дешифрования, обработка Unicode.
2.  **Database Tests:** Проверка схемы БД, CRUD операций, каскадного удаления тегов.
3.  **Logic Tests:** Проверка алгоритмов фильтрации, поиска и генерации паролей.

**Запуск тестов:**
```bash
./gradlew jvmTest --rerun-tasks
```

*Отчет о результатах тестирования доступен в:* `composeApp/build/reports/tests/jvmTest/index.html`
